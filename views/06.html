Paint by Numbers




drawing a polygon



up until now we’ve mostly been drawing rectangles. drawing more complicated shapes ( or "paths" ) requires a few more lines of code (1) ctx.beginPath() to start a new path, (2) ctx.moveTo(x,y) to specify where you want to start your path (3) ctx.lineTo(x,y) to draw a line to a new point ( create as many of these points as you want ), (4) then you can either stroke your path with ctx.stroke() or fill it with ctx.fill() or both, and finally (5) ctx.closePath()








drawing a circle






to draw an arc or circle use the ctx.arc() method instead of the lineTo method, the arc method takes a few parameters explained in the diagram above






four lines of code and five parameters is a lot to write especially if we want to draw a few circles. and if we add more lines to our circle ( like change the color or add a fill ) it'll get even longer! but we can minimize our code by creating a custom circle function like this one. in between our custom circle function’s parenthesis we can declare variables to use as arguments, three useful ones might be x, y and radius. then we can copy+paste our the code that creates a circle in between our custom function’s curly brackets and replace the hardcoded x,y and radius values with are argument variables. now we can reuse this as many times as we want







more loops



now let’s say we want to draw tons of circles ( maybe dozens maybe hundreds! ) that’s still gonna be a lot of code because we’ll have to call our circle() function tons of times. one way to avoid having to type the same thing over and over again is to use a kind of loop called a for loop





in the example below we create a for loop that starts an increment variable ( i ) at 0, and it increments it by 40 each loop. the loop is set to keep going until it reaches 400, and because 40 goes into 400 ten times, the loop should run 10 times and we should get 10 circles on the screen.




another kind of programmable loop is called a while loop. inside the while loop’s parenthesis you only specify a conditional, the incrementing of variables is something you hand inside the loop itself. the following example does the same as the example above but implementing a while loop instead of a for loop 


var i = 0;
while (i < 400) {
    i+=40;
    circle( i, i, 50 );
};








creating custom shapes





if you prefer to trace over an image, you can load one up into the canvas below by pasting an image URL into the input field and clicking on "add trace image". then follow the short instructions for generating canvas code from tracing the image.





some custom shapes can get so long ( like the one above ) that it helps to consolidate them into a custom function just like we did with our circle. below are some steps you can take to create your own custom shape function.



step 0: we’ll start by creating a new function to contain our custom shape, you can call it whatever you want. I’ll call mine “chiStar” because it’s the star from the Chicago flag. since we’re creating our own custom function we get to specify which arguments it takes. three useful parameters for our shape might be an x coordinate, y coordinate and rotation. we can declare those variables by listing them between the function’s parentheses. 


function chiStar( x, y, rotation ){
	
}


step 1: next copy+paste all the code from your custom shape in between the functions curly brackets ( make sure to keep your code clean and readable by indenting ).

next we’ll try calling our new function underneath with the x, y and rotation arguments set. you’ll notice that our function is ignoring our parameters, that’s because we’re not using the corresponding variables inside our custom function yet. 


function chiStar( x, y, rotation ){
    ctx.fillStyle = "rgba(255,0,0,0.3)";
    ctx.strokeStyle = "rgba(0,150,255,0.3)";
    ctx.lineWidth = 5;
    ctx.beginPath();
    ctx.moveTo(145,42);
    ctx.lineTo(165,103);
    ctx.lineTo(228,89);
    ctx.lineTo(186,138);
    ctx.lineTo(226,184);
    ctx.lineTo(164,171);
    ctx.lineTo(145,232);
    ctx.lineTo(124,171);
    ctx.lineTo(61,184);
    ctx.lineTo(104,138);
    ctx.lineTo(61,89);
    ctx.lineTo(124,103);
    ctx.closePath();
    ctx.fill();
    ctx.stroke();
}

chiStar(100,100,0);


translation




step 2: let’s use our x,y arguments, but rather than using them to draw our lines/shapes we’ll use them to move the entire canvas. by default our drawing context starts at (0,0) but we can move our canvas by using the ctx.translate() method which takes x,y coordinates 


function chiStar( x, y, rotation ){
	
    ctx.translate(x,y);

    ctx.fillStyle = "rgba(255,0,0,0.3)";
    // abreviated for clarity ...
    ctx.beginPath();
    ctx.moveTo(145,42);
    ctx.lineTo(165,103);
    // abreviated for clarity ...
    ctx.stroke();
}

chiStar(100,100,0);


next we’ll reposition our entire shape so that the center of the shape is at our new (0,0) position. we’ll do this by creating two new variables ( let’s call them dx and dy ) and then adding dx to every x value in our moveTo/lineTo coordinates and adding dy to every y value. you’ll probably have to play a little bit with what values to set your dx,dy 


function chiStar( x, y, rotation ){
	
    ctx.translate(x,y);

    var dx = -145;
    var dy = -137;

    ctx.fillStyle = "rgba(255,0,0,0.3)";
    // abreviated for clarity ...
    ctx.beginPath();
    ctx.moveTo(145+dx,42+dy);
    ctx.lineTo(165+dx,103+dy);
    // abreviated for clarity ...
    ctx.stroke();
}

chiStar(100,100,0);


rotation




now that we’ve got our shape centered on the (0,0) coordinate of our translated canvas we can rotate it around the center! we do this by using the ctx.rotate() method. the rotate method takes a value in radians as an argument ( the same way we declared the start and end angles when drawing our circles ). but if you prefer to specify rotation in degrees you can use the Math.degreesToRads() to convert the rotation argument to take degrees instead. 


function chiStar( x, y, rotation ){

    ctx.translate(x,y);

    rotation = Math.degreesToRads( rotation );
    ctx.rotate( rotation );

    var dx = -145;
    var dy = -137;

    ctx.fillStyle = "rgba(255,0,0,0.3)";
    // abreviated for clarity ...
    ctx.beginPath();
    ctx.moveTo(145+dx,42+dy);
    ctx.lineTo(165+dx,103+dy);
    // abreviated for clarity ...
    ctx.stroke();
}

chiStar(100,100,0);


now we’ve got a custom shape! there’s only one last edit we need to make to our function. we need to restore the canvas to it’s original state at the original (0,0) coordinate so everything else we draw doesn’t come out wrong. In order to relegate our canvas state translations to our custom function alone we need to add ctx.save() at the top of our function and ctx.restore() at the bottom. 


function chiStar( x, y, rotation ){
    ctx.save(); // save original state

    ctx.translate(x,y);

    rotation = Math.degreesToRads( rotation );
    ctx.rotate( rotation );

    var dx = -145;
    var dy = -137;

    ctx.fillStyle = "rgba(255,0,0,0.3)";
    // abreviated for clarity ...
    ctx.beginPath();
    ctx.moveTo(145+dx,42+dy);
    ctx.lineTo(165+dx,103+dy);
    // abreviated for clarity ...
    ctx.stroke();

    ctx.restore(); // restore original state
}

chiStar(100,100,0);
